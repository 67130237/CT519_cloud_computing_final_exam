หน้าแรก เกี่ยวกับฉัน แนวทางวิจัย จัดการอ้างอิง

ระบบ Agentic AI สำหรับการสืบค้นและวิเคราะห์ Microservices Log ด้วยเทคนิค Agentic AI, Model Context Protocol และ ElasticSearch Tooling

บทที่ 1 บทนำ

1.1 ความเป็นมาและความสำคัญของปัญหา

ในยุคของระบบซอฟต์แวร์แบบกระจาย (Distributed Systems) และสถาปัตยกรรมไมโครเซอร์วิส (Microservices Architecture)...

แนวทาง Agentic AI ซึ่งผสานความสามารถของโมเดลภาษา (LLM) กับการเชื่อมโยง tool เชิงบริบท เช่น ElasticSearch...

งานวิจัยนี้จึงมุ่งเน้นการพัฒนาระบบ Agentic AI ที่สามารถสืบค้น วิเคราะห์ และสรุปข้อมูลจาก log ของระบบไมโครเซอร์วิส...

sequence_diagram

1.2 ขอบเขตของการวิจัย

  • พัฒนาระบบ Agentic AI ที่ทำงานร่วมกับ LLM ผ่าน MCP Protocol
  • รองรับ use case เช่น error tracing, context-aware log search, summarization
  • ไม่แก้ไข ElasticSearch หรือ retrain LLM
  • ใช้ log จำลองใกล้เคียงกับ production

1.3 วัตถุประสงค์ของการวิจัย

  • พัฒนาระบบ Agentic AI ที่เชื่อม ElasticSearch ผ่าน MCP
  • ประเมินความสามารถ LLM ในการวิเคราะห์ log
  • เปรียบเทียบความแม่นยำ ความเร็ว ความเข้าใจ กับวิธีดั้งเดิม
  • สร้างต้นแบบใช้งานใน DevOps และ Incident Response

1.4 ประโยชน์ที่คาดว่าจะได้รับ

  • ลดภาระ System Engineer
  • เพิ่มความแม่นยำและความเร็วในการตรวจจับปัญหา
  • เพิ่ม natural language insight ให้ระบบ observability
  • วางรากฐาน Agentic AI สำหรับ IT Ops

1.5 การวางแผนดำเนินการ

การดำเนินการจะถูกแบ่งออกเป็นระยะเวลา 4 เดือน:

  • เดือน 1: ศึกษา Agentic AI, MCP, ElasticSearch
  • เดือน 2: ออกแบบ MCP Server + Tool Interface
  • เดือน 3: เตรียม log dataset และทดสอบ LLM
  • เดือน 4: วิเคราะห์ผล + เขียนเล่ม
